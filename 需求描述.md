# 需求描述

百万级数据导出后端服务跨语言性能基准测试，所有服务使用 Docker Compose 容器编排启动。

---

## 1. 数据库设计

### 1.1 技术选型
- **数据库**：PostgreSQL 17+（尽量使用最新版本）
- **部署方式**：Docker 镜像运行
- **容器配置**：提前优化内存、CPU、磁盘等参数，确保数据库性能最优

### 1.2 表结构设计
模拟业务场景为**订单记录表**，单表 30 个字段左右。

#### 字段清单

| 序号 | 字段名 | 类型 | 长度 | 说明 | 生成规则 |
|------|--------|------|------|------|----------|
| 1 | order_id | BIGINT | - | 订单ID，主键 | 自增序列，从 1 开始 |
| 2 | order_no | VARCHAR | 32 | 订单编号，唯一索引 | 格式：ORD + 时间戳 + 6位随机数 |
| 3 | user_id | BIGINT | - | 用户ID，外键 | 范围：1 ~ 20000000 |
| 4 | user_name | VARCHAR | 64 | 用户姓名 | 随机中文姓名，2-4 个汉字 |
| 5 | user_phone | VARCHAR | 11 | 用户手机号 | 符合手机号规范：13/14/15/16/17/18/19 开头 |
| 6 | user_id_card | VARCHAR | 18 | 用户身份证号 | 符合身份证格式，含生日和校验位 |
| 7 | user_email | VARCHAR | 128 | 用户邮箱 | user_{user_id}@example.com |
| 8 | user_address | VARCHAR | 256 | 用户收货地址 | 随机省市区地址 |
| 9 | product_id | BIGINT | - | 商品ID | 范围：1 ~ 100000 |
| 10 | product_name | VARCHAR | 128 | 商品名称 | 随机商品名前缀 + 规格 |
| 11 | product_category | VARCHAR | 64 | 商品分类 | 预定义分类池 |
| 12 | product_price | DECIMAL | 10,2 | 商品单价 | 范围：0.01 ~ 9999.99 |
| 13 | quantity | INT | - | 购买数量 | 范围：1 ~ 99 |
| 14 | total_amount | DECIMAL | 12,2 | 订单总金额 | = product_price * quantity |
| 15 | discount_amount | DECIMAL | 12,2 | 优惠金额 | 范围：0 ~ total_amount * 0.3 |
| 16 | pay_amount | DECIMAL | 12,2 | 实付金额 | = total_amount - discount_amount |
| 17 | order_status | VARCHAR | 16 | 订单状态 | 待支付/已支付/已取消/已退款/已完成 |
| 18 | payment_method | VARCHAR | 16 | 支付方式 | 支付宝/微信/银行卡/其他 |
| 19 | payment_time | TIMESTAMP | - | 支付时间 | 在下单时间之后，允许为空 |
| 20 | order_source | VARCHAR | 32 | 订单来源 | APP/小程序/Web/H5/线下 |
| 21 | shipping_address | VARCHAR | 512 | 收货地址 | 随机省市区详细地址 |
| 22 | receiver_name | VARCHAR | 64 | 收货人姓名 | 随机中文姓名 |
| 23 | receiver_phone | VARCHAR | 11 | 收货人手机号 | 符合手机号规范 |
| 24 | logistics_no | VARCHAR | 32 | 物流单号 | 格式：SF/YT/YD + 12位数字，允许为空 |
| 25 | delivery_time | TIMESTAMP | - | 发货时间 | 在支付时间之后，允许为空 |
| 26 | complete_time | TIMESTAMP | - | 完成时间 | 在发货时间之后，允许为空 |
| 27 | remark | VARCHAR | 512 | 订单备注 | 随机生成，可为空 |
| 28 | created_at | TIMESTAMP | - | 创建时间（下单时间），索引 | 范围：2020-01-01 ~ 2024-12-31 |
| 29 | updated_at | TIMESTAMP | - | 更新时间 | >= created_at |
| 30 | is_deleted | SMALLINT | - | 软删除标记 | 0-未删除，1-已删除，默认 0 |

### 1.3 索引设计

| 索引名称 | 索引类型 | 索引字段 | 说明 |
|----------|----------|----------|------|
| idx_order_id | PRIMARY | order_id | 主键索引 |
| uk_order_no | UNIQUE | order_no | 订单编号唯一索引 |
| idx_user_id | BTree | user_id | 用户ID索引 |
| idx_created_at | BTree | created_at | 下单时间索引，范围查询 |
| idx_order_status | BTree | order_status | 订单状态索引 |
| idx_total_amount | BTree | total_amount | 金额索引，范围查询 |
| idx_user_created | BTree | (user_id, created_at) | 复合索引，用户+时间联合查询 |
| idx_status_created | BTree | (order_status, created_at) | 复合索引，状态+时间联合查询 |

### 1.4 数据生成逻辑

#### 1.4.1 初始数据生成
- **数据量**：2000 万条记录
- **生成方式**：使用数据生成工具（如 pg_bulkload、COPY 命令或自定义程序）
- **时间分布**：
  - 2020年：10%
  - 2021年：15%
  - 2022年：20%
  - 2023年：25%
  - 2024年：30%

#### 1.4.2 增量数据生成
- 支持按时间增量生成（如每日新增订单）
- 支持指定时间范围重新生成
- 提供数据清理脚本（按时间范围删除）

#### 1.4.3 状态分布

| 订单状态 | 占比 |
|----------|------|
| 待支付 | 15% |
| 已支付 | 50% |
| 已取消 | 15% |
| 已退款 | 5% |
| 已完成 | 15% |

## 2. 前端服务

### 2.1 技术栈
- **框架**：Vue.js（需要打包部署）
- **状态管理**：Pinia / Vuex
- **HTTP 客户端**：Axios
- **UI 组件**：Element Plus / Ant Design Vue

### 2.2 功能需求

#### 2.2.1 数据查询
- 查询字段（3-5 个常用字段）：下单时间、订单状态、订单金额、用户ID、订单编号
- 支持多条件组合筛选
- 分页展示（默认每页 20 条）

#### 2.2.2 数据导出
- 支持筛选条件导出（时间范围、订单状态、金额范围、用户ID/地区）
- 支持直接选择数据量导出（100万、200万、500万、1000万、2000万）
- 支持导出格式：CSV（默认）、Excel（xlsx）
- 导出方式：同步导出、异步导出、流式导出（三种都支持）

#### 2.2.3 异步任务管理
- 异步导出返回任务ID
- 支持 SSE 实时推送任务进度
- 支持页面刷新查询任务状态
- 任务完成后可下载文件

### 2.3 安全要求
- **不做登录认证**
- **不允许公开访问**：前端通过 API Key 进行身份验证
- API Key 存储在环境变量中，前端请求时通过 Header 传递

## 4. API 接口设计

### 4.1 通用规范

- **认证方式**：API Key（通过 `X-API-Key` Header 传递）
- **响应格式**：JSON
- **字符编码**：UTF-8

### 4.2 接口列表

#### 4.2.1 查询接口

```
GET /api/v1/orders
```

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| page_size | int | 否 | 每页条数，默认 20 |
| start_time | string | 否 | 开始时间（yyyy-MM-dd） |
| end_time | string | 否 | 结束时间（yyyy-MM-dd） |
| status | string | 否 | 订单状态 |
| min_amount | decimal | 否 | 最小金额 |
| max_amount | decimal | 否 | 最大金额 |
| user_id | string | 否 | 用户ID |
| order_no | string | 否 | 订单编号 |

**响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "total": 20000000,
    "page": 1,
    "page_size": 20,
    "items": [...]
  }
}
```

#### 4.2.2 同步导出接口

```
POST /api/v1/exports/sync
```

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| format | string | 否 | 导出格式：csv（默认）、xlsx |
| limit | int | 否 | 导出数量，不传则按筛选条件 |
| conditions | object | 否 | 筛选条件（同查询接口） |

**响应**：文件流（Content-Type: application/octet-stream）

#### 4.2.3 异步导出接口

```
POST /api/v1/exports/async
```

**请求参数**：同同步导出接口

**响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "uuid-xxx-xxx",
    "status": "pending"
  }
}
```

#### 4.2.4 任务状态查询接口

```
GET /api/v1/exports/tasks/{task_id}
```

**响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "uuid-xxx-xxx",
    "status": "processing",
    "progress": 45,
    "created_at": "2024-01-01T10:00:00Z",
    "completed_at": null,
    "download_url": null
  }
}
```

#### 4.2.5 任务进度推送（SSE）

```
GET /api/v1/exports/sse/{task_id}
```

**SSE 事件**：
```
event: progress
data: {"progress": 50}

event: completed
data: {"download_url": "/api/v1/exports/download/token-xxx"}

event: failed
data: {"error": "error message"}
```

#### 4.2.6 文件下载接口

```
GET /api/v1/exports/download/{token}
```

**响应**：文件流

#### 4.2.7 流式导出接口

```
POST /api/v1/exports/stream
```

**请求参数**：同同步导出接口

**响应**：SSE 流式数据（Content-Type: text/event-stream）

## 5. 测试方案

### 5.1 测试方式
所有服务均实现以下三种导出方式：
1. **同步导出** - 用户发起请求后一直等待，直到文件生成完成并下载
2. **异步导出** - 用户发起请求后立即返回任务ID，后台处理完成后通知用户下载
3. **流式导出** - 边查询边写入响应流，用户逐步接收数据

### 5.2 并发级别
- 单用户测试
- 低并发测试（5-10 并发）
- 中并发测试（20-50 并发）
- 高并发测试（100+ 并发）

### 5.3 数据量档位
100万、200万、500万、1000万、2000万

### 5.4 筛选条件（支持多条件组合）
- 时间范围（下单时间）
- 订单状态（待支付/已支付/已取消/已退款/已完成）
- 金额范围
- 用户ID/地区

### 5.5 压测工具
wrk / wrk2

### 5.6 结果输出
可视化图表（自动绘图）
